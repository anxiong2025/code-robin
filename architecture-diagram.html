QueryEngine — Tool-call Loop State Machine

1

压缩管道 Compression Pipeline

内容替换 → Snip → Microcompact → Context Collapse → Autocompact

2

callModel() 流式循环

流式接收 LLM 响应 → 收集 tool_use blocks → 流式工具执行器并行执行工具 → 实时轮询已完成结果

3

后处理与恢复

prompt-too-long → 响应式压缩  |  max_output_tokens → 3 级恢复  |  stop hooks 检查

4

工具结果收集

收集所有工具执行结果，构建 tool_result 消息

5

附件注入

注入记忆（memdir）、技能发现结果、系统提示更新

6

循环判断

turnCount++ → 限制检查 → needsFollowUp? → true: 带工具结果继续循环  |  false: 退出生成最终结果

Key Architecture Patterns

编译期特性开关

bun:bundle 在构建时消除未启用的特性分支，整个模块被移除。

已知开关：PROACTIVE, KAIROS, BRIDGE_MODE, DAEMON, VOICE_MODE, COORDINATOR_MODE

懒加载

OpenTelemetry、gRPC、Analytics 等重模块通过动态 import() 按需加载，加速冷启动

Memoization

系统/用户上下文在会话期间通过 memoize() 缓存，避免重复计算系统提示等

IDE 桥接

src/bridge/ 与 VS Code / JetBrains 双向通信：JWT 认证、REPL 桥接、权限委托

持久化记忆

src/memdir/ 文件级记忆系统，跨会话持久化用户偏好、项目上下文、反馈

流式工具执行

工具在 LLM 流式输出期间并行执行，不等待响应完成后串行执行，最大化吞吐
